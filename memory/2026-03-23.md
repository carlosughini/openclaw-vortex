### ❌ Falhas Críticas e Lições Aprendidas (para evitar no futuro):
- **Integração Google Calendar/Drive via `gog CLI`:**
    - **Problema:** Falha persistente na autenticação. Causado por:
        - Tentativa inicial de instalação via `npm` (incorreto, `gog` é um projeto Go).
        - Erros na compreensão do fluxo `gog auth login` e `redirect_uri` (`http://localhost` vs `urn:ietf:wg:oauth:2.0:oob`) para ambiente de VPS.
        - Não uso adequado de `gog auth credentials`.
    - **Lição Aplicada:** Para instalações Go, `golang` deve estar instalado. Para autenticação remota em CLI, o fluxo Out-of-Band (OOB) é essencial, mas a ferramenta requer que o `redirect_uris` no arquivo de credenciais do app seja `urn:ietf:wg:oauth:2.0:oob`, ou que o comando `gog auth credentials` seja usado de forma precisa com um cliente `Desktop app`. O ideal é um fluxo manual via `--remote --step 1` e `--remote --step 2` com a URL de redirecionamento fornecida pelo usuário.
- **Integração Bitwarden CLI (`bw`):**
    - **Problema:** Falha persistente na instalação (`command not found`) e na criação de itens.
        - `bw` instalado via `snap` não estava no `PATH` automaticamente.
        - Erros de sintaxe ao tentar criar itens (`bw create item`) via JSON e Master Password. A utilização de `BW_SESSION` e `BW_PASSWORD` como ambiente ou flags e a formatação Base64 do JSON são cruciais.
    - **Lição Aplicada:** O executável `bw` do Snap está em `/snap/bin/bw`. Para comandos não interativos, é necessário usar variáveis de ambiente como `BW_PASSWORD` e passar o JSON corretamente, muitas vezes Base64 encoded ou através de um arquivo. Operações sensíveis pedem a Master Password novamente.
- **Configurações do OpenClaw via `config.patch`:**
    - **Problema:** Erros repetitivos na especificação do `path` e na estrutura do `raw` JSON, causando `Unrecognized key` e `SyntaxError`.
    - **Lição Aplicada:** Sempre consultar `gateway.config.schema.lookup` para o caminho exato. O `raw` para `config.patch` precisa ser um objeto JSON aninhado corretamente, e `memorysearch.experimental.sessionMemory` é um booleano, não um objeto.
